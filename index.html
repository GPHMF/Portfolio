FULCRUM - Provider Wellness Platform

PROTOTYPE
Live Deployment GitHub Repository

Detailed project information available at the GitHub repository.

Best viewed on desktop/laptop/tablet (1024px+). Mobile optimization in progress.

FULCRUM homepage showing the main sections and provider role selection.

A wellness resource addressing healthcare provider burnout (46% of providers report burnout) by organizing evidence-based support strategies across 8 different provider roles.

The platform demonstrates how a healthcare system might organize and deploy provider wellness infrastructure at scale; from individual coping strategies to organizational leadership interventions.

STATUS: Prototype in active development

Development Status: Portfolio proof-of-concept (desktop version complete)

Current Focus: Clinical examination preparation (Step 3, December 2-9)

Timeline to Further Development: Mobile and accessibility refinement resuming post-exam

Core architecture complete and functional; polishing and feature enhancements will resume after medical board examination

Key Features

  • 8 Provider Roles: Physicians, Residents, Nurse Practitioners, Physician Assistants, Nurses, Pharmacists, Therapists, Medical Assistants
  • Role-Specific Resources: Customized stress management and wellness strategies for each provider type
  • Crisis Support Center: 25+ resources (988 Lifeline, Crisis Text Line, specialty services)
  • ROI Calculator: Estimates financial impact of wellness initiatives
  • Organizational Strategies: 68 evidence-based interventions for healthcare leadership
  • Search: Unified search across 3,000+ content items with advanced relevance scoring (along with scoped search in the provider-specific and organizational strategies sections)
  • Dark/Light Theme: Automatic system preference detection with option for manual override (with theme persistence across site reloads)

Technical Stack

  • HTML5 for content
  • CSS3 for UI styling
  • Vanilla JavaScript (ES6+) for UX and features
  • Chart.js for visualization charts
  • No external frameworks
  • GitHub Pages deployment

Clinical Foundation

  • Evidence-based (peer-reviewed sources)
  • Addresses documented healthcare crisis

Why This Matters

Provider wellness directly affects patient care quality and system safety. This project demonstrates how technology can help organize evidence-based support at healthcare system scale.

What's Next

  • Mobile-responsive design
  • Accessibility refinement (WCAG compliance)
  • Performance optimization
  • Cross-browser testing

Development Approach

My approach to healthcare technology is practical and evidence-based:

1

Solve Real Problems First

Both projects address documented healthcare challenges (burnout, health literacy). Technology should solve specific clinical problems, not create new ones.

2

Clinical Accuracy is Non-Negotiable

All content is based on peer-reviewed literature. In healthcare, technical excellence without clinical accuracy is worthless.

3

Long-Term Learning

I'm interested in understanding how healthcare systems actually implement technology. What works, what fails, and why. This foundation helps build better solutions.

Interests & Learning

As I develop as a clinician, I'm interested in deepening my understanding of:

  • How healthcare systems successfully implement new technologies
  • Clinical workflow optimization and system-level thinking
  • How to translate clinical problems into effective technology solutions

Beyond application-layer healthcare technology, I'm also exploring foundational AI approaches as a hobbyist interest. I'm curious about:

  • How machine learning can enhance clinical decision-making
  • Feature engineering from clinical datasets
  • The ethics of deploying AI in healthcare settings

I see clinical residency training as the immediate foundation; understanding clinical practice deeply before exploring more advanced technical applications. Additionally, it provides access to institutional resources, clinical data, and mentorship that independent learning cannot replicate.

I'm particularly drawn to residency programs that emphasize:

  • Systems-level thinking about healthcare delivery
  • Quality improvement and workflow optimization
  • Innovation and continuous improvement culture
  • Mentorship in healthcare technology and informatics

These interests drive my project selection; both Fulcrum and the Wellness-Simplified demonstrate practical problem-solving. I'm eager to bring this same approach to learning clinical medicine.
